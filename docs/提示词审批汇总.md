# 提示词审批汇总（多智能体）

## 概览
- 范围：协作与权限、邀请/分享、活动日志、版本回溯与恢复。
- 结论：总体方向批准；需补齐RBAC→ABAC策略、邀请安全与TTL、审计WORM与保留、API幂等/条件更新、UI无障碍细则、代码门禁与性能证据。

## 后端架构（backend-architect-api）
- 批准：RBAC主干+ABAC补充；版本不可变+操作日志可回放；Redis缓存与Kafka解耦；p99≤300ms、≥99.99%可用。
- 需改：
  - 将职能角色映射为平台角色与资源域；补齐`notes/attachments/search/audit/admin`操作集合。
  - 引入ABAC属性（`classification/org_id/ip_range/device_trust/time_window/expires_at/max_views`）。
  - 明确多人编辑冲突策略（优CRDT）、回滚流程与WORM链式哈希。
  - 标准化缓存键/TTL与写后失效一致性等级。

## API架构（api-architect）
- 批准：JWT鉴权+资源Scope；游标分页；Idempotency-Key与ETag/If-Match；错误码子字典；恢复/邀请限流。
- 需改：
  - JWT声明与最小Scope；邀请令牌一次性短TTL（15–60分钟，默认≤72h）；禁止邮箱明文泄露。
  - 变更操作双写审计与`reason`；条件更新统一`412`。
  - 幂等窗口24h与重放标记；日志/版本必须游标分页固定排序；灰度与只读降级。

## 接口测试（api-lifecycle-qa）
- 批准：四阶流程完善；质量出口量化；工具链覆盖。
- 需改：
  - 统一错误码字典、超时SLA、过滤规范、`trace_id`与日志结构；API版本策略。
  - Token旋转与黑名单、IDOR强测、并发控制（Idempotency、ETag）、安全头与文件上传策略。
  - 性能与容量数据、灰度回滚流程、环境一致性校验。

## 前端架构（frontend-architect）
- 批准：React+Vite+SWR+Zustand与路由懒加载；页面与组件划分清晰；性能目标合理。
- 需改：
  - 增设`security/a11y/devops/i18n/analytics`职责与RACI；交付物模板与阈值告警。
  - 冲突解决与回滚规范、国际化与数据度量方案。

## UI交互（ui-chain-expert）
- 批准：覆盖邀请/权限/模式/日志/版本全流程；视觉与可达规则吻合。
- 需改：
  - 权限矩阵与授权人；邀请渠道/有效期/批量与备注；并发协作规则。
  - 事件类型字典与留存、版本命名与比较维度、恢复生成新版本的规则；审批与原因必填。

## UI质量与可访问性（ui-quality-inspector）
- 批准：容差阈值清晰（对比度/像素/动效）；SLA与证据工件完备。
- 需改：
  - 非文本对比度与焦点可见性、Tab顺序与跳转主内容、ARIA最小集合与冲突禁止。
  - 缩放与高对比度模式回归、证据命名与缺陷台账条款编号、老年模式与色盲安全。

## 合规（product-compliance-auditor）
- 条件批准：角色化提示词不直接采集敏感信息；研究边界降低风险。
- 需改：
  - 数据最小化矩阵与DPIA；邀请TTL≤72h/一次性/即时撤销+通知；关键事件WORM与保留≥24月证据。
  - DSAR入口与30天SLA；跨境SCC/TIA与本地备案；日志脱敏与分权；异常邀请/导出告警阈值。

## 代码质量统筹（code-quality-coordinator）
- 批准：质量红线与冻结策略、评审闭环与门禁方向正确。
- 需改：
  - 版本锁定矩阵与移除`^`漂移、分支保护与PR模板强制字段、静态阈值与CI阻断条件细化。

## 性能优化（performance-optimizer）
- 结论：暂不批准，需要实测证据；给出触发标准与优化建议、监控阈值与验证流程。
- 需改：补充前端RUM、后端APM、数据库与缓存证据，以及并发500/800/1000压测报告。
